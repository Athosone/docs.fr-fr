### <a name="calling-attributegetcustomattributes-on-an-indexer-property-no-longer-throws-ambiguousmatchexception-if-the-ambiguity-can-be-resolved-by-indexs-type"></a>L’appel d’Attribute.GetCustomAttributes sur une propriété d’indexeur ne lève plus d’exception AmbiguousMatchException si l’ambiguïté peut être résolue par type d’index

|   |   |
|---|---|
|Détails|Avant le .NET Framework 4.6, l’appel de <code>GetCustomAttribute(s)</code> sur une propriété d’indexeur qui différait d’une autre propriété uniquement par son type d’index entraînait une <xref:System.Reflection.AmbiguousMatchException?displayProperty=name>. À compter de la version 4.6 du .NET Framework, les attributs de la propriété sont correctement retournés.|
|Suggestion|Notez que GetCustomAttribute(s) fonctionne plus fréquemment à présent. Si une application comptait sur l’exception <xref:System.Reflection.AmbiguousMatchException?displayProperty=name>, utilisez la réflexion pour rechercher explicitement plusieurs indexeurs.|
