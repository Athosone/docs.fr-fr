<span data-ttu-id="0e300-103">Un assembly est l’unité de déploiement et d’identité pour les programmes de code managé.</span><span class="sxs-lookup"><span data-stu-id="0e300-103">An assembly is the unit of deployment and identity for managed code programs.</span></span> <span data-ttu-id="0e300-104">Bien que les assemblys peuvent s’étendre sur un ou plusieurs fichiers, en général, un assembly effectue un mappage avec une DLL.</span><span class="sxs-lookup"><span data-stu-id="0e300-104">Although assemblies can span one or more files, typically an assembly maps one-to-one with a DLL.</span></span> <span data-ttu-id="0e300-105">Par conséquent, cette section décrit les conventions d’affectation de noms de seul DLL peuvent alors être mappées aux conventions d’affectation de noms d’assembly.</span><span class="sxs-lookup"><span data-stu-id="0e300-105">Therefore, this section describes only DLL naming conventions, which then can be mapped to assembly naming conventions.</span></span>  
  
 <span data-ttu-id="0e300-106">**✓ DO** choisissez des noms pour vos DLL d’assembly qui suggèrent des grands segments de fonctionnalités, telles que System.Data.</span><span class="sxs-lookup"><span data-stu-id="0e300-106">**✓ DO** choose names for your assembly DLLs that suggest large chunks of functionality, such as System.Data.</span></span>  
  
 <span data-ttu-id="0e300-107">Noms d’assembly et les DLL n’êtes pas obligé de correspondent aux noms d’espace de noms, mais il est raisonnable de suivre le nom de l’espace de noms lorsque vous nommez des assemblys.</span><span class="sxs-lookup"><span data-stu-id="0e300-107">Assembly and DLL names don’t have to correspond to namespace names, but it is reasonable to follow the namespace name when naming assemblies.</span></span> <span data-ttu-id="0e300-108">Une règle empirique consiste à nommer la DLL basée sur le préfixe commun des espaces de noms contenus dans l’assembly.</span><span class="sxs-lookup"><span data-stu-id="0e300-108">A good rule of thumb is to name the DLL based on the common prefix of the namespaces contained in the assembly.</span></span> <span data-ttu-id="0e300-109">Par exemple, un assembly avec deux espaces de noms, `MyCompany.MyTechnology.FirstFeature` et `MyCompany.MyTechnology.SecondFeature`, peut être appelée `MyCompany.MyTechnology.dll`.</span><span class="sxs-lookup"><span data-stu-id="0e300-109">For example, an assembly with two namespaces, `MyCompany.MyTechnology.FirstFeature` and `MyCompany.MyTechnology.SecondFeature`, could be called `MyCompany.MyTechnology.dll`.</span></span>  
  
 <span data-ttu-id="0e300-110">**✓ CONSIDER** nommer les DLL selon le modèle suivant :</span><span class="sxs-lookup"><span data-stu-id="0e300-110">**✓ CONSIDER** naming DLLs according to the following pattern:</span></span>

 <span data-ttu-id="0e300-111">où `<Component>` contient une ou plusieurs clauses séparés par des points.</span><span class="sxs-lookup"><span data-stu-id="0e300-111">where `<Component>` contains one or more dot-separated clauses.</span></span> <span data-ttu-id="0e300-112">Exemple :</span><span class="sxs-lookup"><span data-stu-id="0e300-112">For example:</span></span>
