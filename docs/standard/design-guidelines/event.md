# <a name="event-design"></a><span data-ttu-id="23c52-102">Conception d'événements</span><span class="sxs-lookup"><span data-stu-id="23c52-102">Event Design</span></span>
<span data-ttu-id="23c52-103">Les événements sont le plus couramment utilisé formulaire des rappels (constructions qui permettent l’infrastructure d’appeler dans le code utilisateur).</span><span class="sxs-lookup"><span data-stu-id="23c52-103">Events are the most commonly used form of callbacks (constructs that allow the framework to call into user code).</span></span> <span data-ttu-id="23c52-104">Autres mécanismes de rappel incluent les membres en prenant les délégués, les membres virtuels et basée sur l’interface de plug-ins. Données à partir des études de convivialité indiquent que la majorité des développeurs sont plus à l’aise avec les événements qu’ils sont à l’aide des autres méthodes de rappel.</span><span class="sxs-lookup"><span data-stu-id="23c52-104">Other callback mechanisms include members taking delegates, virtual members, and interface-based plug-ins. Data from usability studies indicate that the majority of developers are more comfortable using events than they are using the other callback mechanisms.</span></span> <span data-ttu-id="23c52-105">Les événements sont bien intégrées à Visual Studio et de nombreux langages.</span><span class="sxs-lookup"><span data-stu-id="23c52-105">Events are nicely integrated with Visual Studio and many languages.</span></span>  
  
 <span data-ttu-id="23c52-106">Il est important de noter qu’il existe deux groupes d’événements : événements déclenchés avant un état des modifications système, appelé pré-événements et les événements déclenchés après un état change, appelés post événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-106">It is important to note that there are two groups of events: events raised before a state of the system changes, called pre-events, and events raised after a state changes, called post-events.</span></span> <span data-ttu-id="23c52-107">Voici un exemple d’un événement pre `Form.Closing`, qui est déclenché avant la fermeture d’un formulaire.</span><span class="sxs-lookup"><span data-stu-id="23c52-107">An example of a pre-event would be `Form.Closing`, which is raised before a form is closed.</span></span> <span data-ttu-id="23c52-108">Voici un exemple d’un post-événement `Form.Closed`, qui est déclenché après qu’un formulaire est fermé.</span><span class="sxs-lookup"><span data-stu-id="23c52-108">An example of a post-event would be `Form.Closed`, which is raised after a form is closed.</span></span>  
  
 <span data-ttu-id="23c52-109">**✓ DO** utiliser le terme « r » pour les événements plutôt que « fire » ou « déclenche ».</span><span class="sxs-lookup"><span data-stu-id="23c52-109">**✓ DO** use the term "raise" for events rather than "fire" or "trigger."</span></span>  
  
 <span data-ttu-id="23c52-110">**✓ DO** utiliser <xref:System.EventHandler%601?displayProperty=nameWithType> au lieu de créer manuellement les nouveaux délégués à utiliser en tant que gestionnaires d’événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-110">**✓ DO** use <xref:System.EventHandler%601?displayProperty=nameWithType> instead of manually creating new delegates to be used as event handlers.</span></span>  
  
 <span data-ttu-id="23c52-111">**✓ CONSIDER** à l’aide d’une sous-classe de <xref:System.EventArgs> comme argument d’événement, sauf si vous êtes absolument que l’événement n’est jamais nécessaire de transporter des données à l’événement de gestion de la méthode, auquel cas vous pouvez utiliser la `EventArgs` taper directement.</span><span class="sxs-lookup"><span data-stu-id="23c52-111">**✓ CONSIDER** using a subclass of <xref:System.EventArgs> as the event argument, unless you are absolutely sure the event will never need to carry any data to the event handling method, in which case you can use the `EventArgs` type directly.</span></span>  
  
 <span data-ttu-id="23c52-112">Si vous envoyez un à l’aide de l’API `EventArgs` directement, vous ne serez jamais en mesure d’ajouter des données à effectuer avec l’événement sans perdre la compatibilité.</span><span class="sxs-lookup"><span data-stu-id="23c52-112">If you ship an API using `EventArgs` directly, you will never be able to add any data to be carried with the event without breaking compatibility.</span></span> <span data-ttu-id="23c52-113">Si vous utilisez une sous-classe, même si initialement vide, vous serez en mesure d’ajouter des propriétés à la sous-classe lorsque nécessaire.</span><span class="sxs-lookup"><span data-stu-id="23c52-113">If you use a subclass, even if initially completely empty, you will be able to add properties to the subclass when needed.</span></span>  
  
 <span data-ttu-id="23c52-114">**✓ DO** utiliser une méthode virtuelle protégée pour déclencher chaque événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-114">**✓ DO** use a protected virtual method to raise each event.</span></span> <span data-ttu-id="23c52-115">Cela s’applique uniquement aux événements non statiques sur des classes unsealed, pas pour les structs, les classes sealed ou des événements statiques.</span><span class="sxs-lookup"><span data-stu-id="23c52-115">This is only applicable to nonstatic events on unsealed classes, not to structs, sealed classes, or static events.</span></span>  
  
 <span data-ttu-id="23c52-116">L’objectif de la méthode est de fournir un moyen d’une classe dérivée gérer l’événement à l’aide d’un remplacement.</span><span class="sxs-lookup"><span data-stu-id="23c52-116">The purpose of the method is to provide a way for a derived class to handle the event using an override.</span></span> <span data-ttu-id="23c52-117">Substitution est un moyen plus flexible, plus rapide et plus naturel pour gérer les événements de classe de base dans les classes dérivées.</span><span class="sxs-lookup"><span data-stu-id="23c52-117">Overriding is a more flexible, faster, and more natural way to handle base class events in derived classes.</span></span> <span data-ttu-id="23c52-118">Par convention, le nom de la méthode doit commencer par « On » et être suivi par le nom de l’événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-118">By convention, the name of the method should start with "On" and be followed with the name of the event.</span></span>  
  
 <span data-ttu-id="23c52-119">La classe dérivée peut choisissez de ne pas appeler l’implémentation de base de la méthode dans son remplacement.</span><span class="sxs-lookup"><span data-stu-id="23c52-119">The derived class can choose not to call the base implementation of the method in its override.</span></span> <span data-ttu-id="23c52-120">Face à cette situation en n’incluant ne pas de traitement dans la méthode qui est requise pour la classe de base fonctionne correctement.</span><span class="sxs-lookup"><span data-stu-id="23c52-120">Be prepared for this by not including any processing in the method that is required for the base class to work correctly.</span></span>  
  
 <span data-ttu-id="23c52-121">**✓ DO** doit accepter un paramètre à la méthode protégée qui déclenche un événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-121">**✓ DO** take one parameter to the protected method that raises an event.</span></span>  
  
 <span data-ttu-id="23c52-122">Le paramètre doit être nommé `e` et doit être tapé sous forme de la classe d’argument d’événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-122">The parameter should be named `e` and should be typed as the event argument class.</span></span>  
  
 <span data-ttu-id="23c52-123">**X DO NOT** passez la valeur null en tant que l’expéditeur lors du déclenchement d’un événement non statique.</span><span class="sxs-lookup"><span data-stu-id="23c52-123">**X DO NOT** pass null as the sender when raising a nonstatic event.</span></span>  
  
 <span data-ttu-id="23c52-124">**✓ DO** passez la valeur null en tant que l’expéditeur lors du déclenchement d’un événement statique.</span><span class="sxs-lookup"><span data-stu-id="23c52-124">**✓ DO** pass null as the sender when raising a static event.</span></span>  
  
 <span data-ttu-id="23c52-125">**X DO NOT** passez la valeur null en tant que le paramètre de données d’événement lors du déclenchement d’un événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-125">**X DO NOT** pass null as the event data parameter when raising an event.</span></span>  
  
 <span data-ttu-id="23c52-126">Vous devez passer `EventArgs.Empty` si vous ne souhaitez pas passer des données à la méthode de gestion d’événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-126">You should pass `EventArgs.Empty` if you don’t want to pass any data to the event handling method.</span></span> <span data-ttu-id="23c52-127">Les développeurs s’attendent à ce paramètre ne pas pour avoir la valeur null.</span><span class="sxs-lookup"><span data-stu-id="23c52-127">Developers expect this parameter not to be null.</span></span>  
  
 <span data-ttu-id="23c52-128">**✓ CONSIDER** le déclenchement d’événements de l’utilisateur final peut annuler.</span><span class="sxs-lookup"><span data-stu-id="23c52-128">**✓ CONSIDER** raising events that the end user can cancel.</span></span> <span data-ttu-id="23c52-129">Cela s’applique uniquement aux pré-événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-129">This only applies to pre-events.</span></span>  
  
 <span data-ttu-id="23c52-130">Utilisez <xref:System.ComponentModel.CancelEventArgs?displayProperty=nameWithType> ou sa sous-classe comme argument d’événement pour autoriser l’utilisateur final d’annuler des événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-130">Use <xref:System.ComponentModel.CancelEventArgs?displayProperty=nameWithType> or its subclass as the event argument to allow the end user to cancel events.</span></span>  
  
### <a name="custom-event-handler-design"></a><span data-ttu-id="23c52-131">Conception de gestionnaire d’événements personnalisés</span><span class="sxs-lookup"><span data-stu-id="23c52-131">Custom Event Handler Design</span></span>  
 <span data-ttu-id="23c52-132">Il existe des cas dans lesquels `EventHandler<T>` ne peut pas être utilisées, comme lorsque le framework doit fonctionner avec les versions antérieures du CLR, qui ne prenait pas en charge des génériques.</span><span class="sxs-lookup"><span data-stu-id="23c52-132">There are cases in which `EventHandler<T>` cannot be used, such as when the framework needs to work with earlier versions of the CLR, which did not support Generics.</span></span> <span data-ttu-id="23c52-133">Dans ce cas, vous devez concevoir et développer un délégué de gestionnaire d’événements personnalisé.</span><span class="sxs-lookup"><span data-stu-id="23c52-133">In such cases, you might need to design and develop a custom event handler delegate.</span></span>  
  
 <span data-ttu-id="23c52-134">**✓ DO** utiliser un type de retour void pour les gestionnaires d’événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-134">**✓ DO** use a return type of void for event handlers.</span></span>  
  
 <span data-ttu-id="23c52-135">Un gestionnaire d’événements peut appeler plusieurs méthodes, éventuellement sur plusieurs objets de gestion d’événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-135">An event handler can invoke multiple event handling methods, possibly on multiple objects.</span></span> <span data-ttu-id="23c52-136">Si les méthodes de gestion des événements ont été autorisées pour retourner une valeur, il y aura plusieurs valeurs de retour pour chaque appel de l’événement.</span><span class="sxs-lookup"><span data-stu-id="23c52-136">If event handling methods were allowed to return a value, there would be multiple return values for each event invocation.</span></span>  
  
 <span data-ttu-id="23c52-137">**✓ DO** utiliser `object` en tant que le type du premier paramètre du Gestionnaire d’événements et l’appeler `sender`.</span><span class="sxs-lookup"><span data-stu-id="23c52-137">**✓ DO** use `object` as the type of the first parameter of the event handler, and call it `sender`.</span></span>  
  
 <span data-ttu-id="23c52-138">**✓ DO** utiliser <xref:System.EventArgs?displayProperty=nameWithType> ou sa sous-classe en tant que le type du deuxième paramètre du Gestionnaire d’événements et l’appeler `e`.</span><span class="sxs-lookup"><span data-stu-id="23c52-138">**✓ DO** use <xref:System.EventArgs?displayProperty=nameWithType> or its subclass as the type of the second parameter of the event handler, and call it `e`.</span></span>  
  
 <span data-ttu-id="23c52-139">**X DO NOT** avoir plus de deux paramètres sur les gestionnaires d’événements.</span><span class="sxs-lookup"><span data-stu-id="23c52-139">**X DO NOT** have more than two parameters on event handlers.</span></span>
