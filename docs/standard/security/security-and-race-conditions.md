## <a name="race-conditions-in-the-dispose-method"></a><span data-ttu-id="8ac19-106">Conditions de concurrence dans la méthode Dispose</span><span class="sxs-lookup"><span data-stu-id="8ac19-106">Race Conditions in the Dispose Method</span></span>  
 <span data-ttu-id="8ac19-107">Si d’une classe **Dispose** (méthode) (pour plus d’informations, consultez [Garbage Collection](../../../docs/standard/garbage-collection/index.md)) est non synchronisé, il est possible que le code de nettoyage à l’intérieur de **Dispose** peut être exécuté plus de une fois, comme illustré dans l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="8ac19-107">If a class's **Dispose** method (for more information, see [Garbage Collection](../../../docs/standard/garbage-collection/index.md)) is not synchronized, it is possible that cleanup code inside **Dispose** can be run more than once, as shown in the following example.</span></span>  
  
```vb  
Sub Dispose()  
    If Not (myObj Is Nothing) Then  
       Cleanup(myObj)  
       myObj = Nothing  
    End If  
End Sub  
```  
  
```csharp  
void Dispose()   
{  
    if (myObj != null)   
    {  
        Cleanup(myObj);  
        myObj = null;  
    }  
}  
```  
  
 <span data-ttu-id="8ac19-108">Étant donné que cela **Dispose** implémentation n’est pas synchronisée, il est possible pour `Cleanup` doit être appelée par tout d’abord un thread, puis un deuxième thread avant `_myObj` a la valeur **null**.</span><span class="sxs-lookup"><span data-stu-id="8ac19-108">Because this **Dispose** implementation is not synchronized, it is possible for `Cleanup` to be called by first one thread and then a second thread before `_myObj` is set to **null**.</span></span> <span data-ttu-id="8ac19-109">S’il s’agit d’un problème de sécurité varie selon que se passe-t-il lorsque le `Cleanup` code s’exécute.</span><span class="sxs-lookup"><span data-stu-id="8ac19-109">Whether this is a security concern depends on what happens when the `Cleanup` code runs.</span></span> <span data-ttu-id="8ac19-110">Un problème majeur avec non synchronisés **Dispose** implémentations implique l’utilisation des descripteurs de ressources tels que les fichiers.</span><span class="sxs-lookup"><span data-stu-id="8ac19-110">A major issue with unsynchronized **Dispose** implementations involves the use of resource handles such as files.</span></span> <span data-ttu-id="8ac19-111">Suppression inappropriée peut entraîner le handle incorrect à utiliser, ce qui entraîne souvent des failles de sécurité.</span><span class="sxs-lookup"><span data-stu-id="8ac19-111">Improper disposal can cause the wrong handle to be used, which often leads to security vulnerabilities.</span></span>

## <a name="race-conditions-with-cached-objects"></a><span data-ttu-id="8ac19-115">Conditions de concurrence avec les objets mis en cache</span><span class="sxs-lookup"><span data-stu-id="8ac19-115">Race Conditions with Cached Objects</span></span>  
 <span data-ttu-id="8ac19-116">Le code qui met en cache les informations de sécurité ou utilise la sécurité d’accès du code [Assert](../../../docs/framework/misc/using-the-assert-method.md) opération peut également être vulnérable aux conditions de concurrence critique si d’autres parties de la classe ne sont pas correctement synchronisées, comme illustré dans l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="8ac19-116">Code that caches security information or uses the code access security [Assert](../../../docs/framework/misc/using-the-assert-method.md) operation might also be vulnerable to race conditions if other parts of the class are not appropriately synchronized, as shown in the following example.</span></span>  
  
```vb  
Sub SomeSecureFunction()  
    If SomeDemandPasses() Then  
        fCallersOk = True  
        DoOtherWork()  
        fCallersOk = False  
    End If  
End Sub  
  
Sub DoOtherWork()  
    If fCallersOK Then  
        DoSomethingTrusted()  
    Else  
        DemandSomething()  
        DoSomethingTrusted()  
    End If  
End Sub  
```  
  
```csharp  
void SomeSecureFunction()   
{  
    if (SomeDemandPasses())   
    {  
        fCallersOk = true;  
        DoOtherWork();  
        fCallersOk = false;  
    }  
}  
void DoOtherWork()   
{  
    if (fCallersOK)   
    {  
        DoSomethingTrusted();  
    }  
    else   
    {  
        DemandSomething();  
        DoSomethingTrusted();  
    }  
}  
```  
  
 <span data-ttu-id="8ac19-117">S’il existe des autres chemins d’accès aux `DoOtherWork` qui peut être appelée à partir d’un autre thread avec le même objet, un appelant non fiable peut ignorer une demande.</span><span class="sxs-lookup"><span data-stu-id="8ac19-117">If there are other paths to `DoOtherWork` that can be called from another thread with the same object, an untrusted caller can slip past a demand.</span></span>  
  
 <span data-ttu-id="8ac19-118">Si votre code met en cache les informations de sécurité, veillez à consulter pour cette vulnérabilité.</span><span class="sxs-lookup"><span data-stu-id="8ac19-118">If your code caches security information, make sure that you review it for this vulnerability.</span></span>
