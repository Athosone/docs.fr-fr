# <a name="compiler-error-cs1952"></a><span data-ttu-id="87235-102">Erreur du compilateur CS1952</span><span class="sxs-lookup"><span data-stu-id="87235-102">Compiler Error CS1952</span></span>

 <span data-ttu-id="87235-104">Le mot clé `__arglist` non pris en charge n’est pas autorisé dans les expressions lambda qui sont compilées en arborescences d’expression.</span><span class="sxs-lookup"><span data-stu-id="87235-104">The unsupported `__arglist` keyword is not allowed in lambda expressions that compile to expression trees.</span></span>

1. <span data-ttu-id="87235-106">Oubliez tout simplement `__arglist`.</span><span class="sxs-lookup"><span data-stu-id="87235-106">Forget that you ever heard of `__arglist`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="87235-107">Exemple</span><span class="sxs-lookup"><span data-stu-id="87235-107">Example</span></span>  
 <span data-ttu-id="87235-108">Le code suivant génère l’erreur CS1952 :</span><span class="sxs-lookup"><span data-stu-id="87235-108">The following code produces CS1952:</span></span>  
  
```csharp  
// cs1952.cs  
using System;  
using System.Linq.Expressions;  
  
class Test  
{  
    public static int M(__arglist)  
    {  
        return 1;  
    }  
  
    static int Main()  
    {  
        Expression<Func<int, int>> f = x => Test.M(__arglist(x)); // CS1952  
        return 1;  
    }  
}  
```
