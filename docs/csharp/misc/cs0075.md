# <a name="compiler-error-cs0075"></a><span data-ttu-id="ed044-102">Erreur du compilateur CS0075</span><span class="sxs-lookup"><span data-stu-id="ed044-102">Compiler Error CS0075</span></span>
<span data-ttu-id="ed044-103">Pour effectuer un cast d'une valeur négative, vous devez la mettre entre parenthèses</span><span class="sxs-lookup"><span data-stu-id="ed044-103">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="ed044-104">Si vous effectuez une conversion de type (transtypage) à partir d’un mot clé qui identifie un type prédéfini, vous n’avez pas besoin de parenthèses.</span><span class="sxs-lookup"><span data-stu-id="ed044-104">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="ed044-105">Dans le cas contraire, vous devez utiliser des parenthèses, car (x) –y ne seront pas considérés comme une expression cast.</span><span class="sxs-lookup"><span data-stu-id="ed044-105">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="ed044-106">D’après la spécification C#, section 7.6.6 :</span><span class="sxs-lookup"><span data-stu-id="ed044-106">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="ed044-107">*À partir de la règle de suppression qui, si x et y sont des identificateurs, (x) y, (suppression, et (x)(-y) sont des expressions de cast, mais (x)-y n’est pas, même si x identifie un type. Toutefois, si x est un mot clé qui identifie un type prédéfini (tel qu’int), les quatre formes sont des expressions de cast (car un tel mot clé ne peut pas être une expression en lui-même).*</span><span class="sxs-lookup"><span data-stu-id="ed044-107">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="ed044-108">Le code suivant génère l’erreur CS0075 :</span><span class="sxs-lookup"><span data-stu-id="ed044-108">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
