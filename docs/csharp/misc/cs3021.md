# <a name="compiler-warning-level-2-cs3021"></a>Avertissement du compilateur (niveau 2) CS3021
'type' n’a pas besoin d’un attribut CLSCompliant, car l’assembly n’a pas d’attribut CLSCompliant  
  
 Cet avertissement se produit si `[CLSCompliant(false)]` apparaît sur une classe dans un assembly qui n’a pas d’un jeu d’attribut CLSCompliant de niveau assembly sur true (autrement dit, la ligne `[assembly: CLSCompliant(true)]`). Étant donné que l’assembly ne se déclare pas comme conforme CLS, il est inutile que son contenu se déclare comme non conforme, puisqu’il est déjà supposé être non conforme. Pour plus d’informations sur la conformité CLS, consultez [indépendance du langage et composants indépendants du langage](../../standard/language-independence.md).
  
 Pour supprimer cet avertissement, supprimez l’attribut ou ajoutez l’attribut de niveau assembly.  
  
## <a name="example"></a>Exemple  
 L’exemple suivant génère l’erreur CS3021 :  
  
```csharp  
// CS3021.cs  
using System;  
// Uncomment the following line to declare the assembly CLS Compliant,  
// and avoid the warning without removing the attribute on the class.  
//[assembly: CLSCompliant(true)]  
  
// Remove the next line to avoid the warning.  
[CLSCompliant(false)]               // CS3021  
public class C  
{  
    public static void Main()  
    {  
    }  
}  
```
