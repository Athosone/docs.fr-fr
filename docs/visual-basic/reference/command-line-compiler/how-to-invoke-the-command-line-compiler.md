<span data-ttu-id="2682b-103">Vous pouvez appeler le compilateur de ligne de commande en tapant le nom de son fichier exécutable dans la ligne de commande, également appelée invite de commande MS-DOS.</span><span class="sxs-lookup"><span data-stu-id="2682b-103">You can invoke the command-line compiler by typing the name of its executable file into the command line, also known as the MS-DOS prompt.</span></span> <span data-ttu-id="2682b-104">Si vous compilez à partir de l’invite de commandes de Windows par défaut, vous devez taper le chemin d’accès complet au fichier exécutable.</span><span class="sxs-lookup"><span data-stu-id="2682b-104">If you compile from the default Windows Command Prompt, you must type the fully qualified path to the executable file.</span></span> <span data-ttu-id="2682b-105">Pour remplacer ce comportement par défaut, vous pouvez utiliser l’invite de commandes développeur pour Visual Studio, ou modifier la variable d’environnement PATH.</span><span class="sxs-lookup"><span data-stu-id="2682b-105">To override this default behavior, you can either use the Developer Command Prompt for Visual Studio, or modify the PATH environment variable.</span></span> <span data-ttu-id="2682b-106">Les deux permettent de compiler à partir de n’importe quel répertoire il suffit de taper le nom du compilateur.</span><span class="sxs-lookup"><span data-stu-id="2682b-106">Both allow you to compile from any directory by simply typing the compiler name.</span></span>

### <a name="to-invoke-the-compiler-using-the-developer-command-prompt-for-visual-studio"></a><span data-ttu-id="2682b-107">Pour appeler le compilateur à l’aide de l’invite de commandes développeur pour Visual Studio</span><span class="sxs-lookup"><span data-stu-id="2682b-107">To invoke the compiler using the Developer Command Prompt for Visual Studio</span></span>

2. <span data-ttu-id="2682b-109">Vous pouvez utiliser l’invite de commandes développeur pour Visual Studio pour accéder au compilateur à partir de n’importe quel répertoire sur votre ordinateur, si Visual Studio est installé.</span><span class="sxs-lookup"><span data-stu-id="2682b-109">You can use the Developer Command Prompt for Visual Studio to access the compiler from any directory on your machine, if Visual Studio is installed.</span></span>  
  
3. <span data-ttu-id="2682b-110">Appeler l’invite de commandes développeur pour Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="2682b-110">Invoke the Developer Command Prompt for Visual Studio.</span></span>  
  
4. <span data-ttu-id="2682b-111">À la ligne de commande, tapez `vbc.exe` *sourceFileName* puis appuyez sur ENTRÉE.</span><span class="sxs-lookup"><span data-stu-id="2682b-111">At the command line, type `vbc.exe` *sourceFileName* and then press ENTER.</span></span>  
  
     <span data-ttu-id="2682b-112">Par exemple, si vous avez stocké votre code source dans un répertoire appelé `SourceFiles`, vous ouvririez l’invite de commandes et le type `cd SourceFiles` pour accéder à ce répertoire.</span><span class="sxs-lookup"><span data-stu-id="2682b-112">For example, if you stored your source code in a directory called `SourceFiles`, you would open the Command Prompt and type `cd SourceFiles` to change to that directory.</span></span> <span data-ttu-id="2682b-113">Si le répertoire contenait un fichier source nommé `Source.vb`, vous pouvez le compiler en tapant `vbc.exe Source.vb`.</span><span class="sxs-lookup"><span data-stu-id="2682b-113">If the directory contained a source file named `Source.vb`, you could compile it by typing `vbc.exe Source.vb`.</span></span>

     <span data-ttu-id="2682b-116">Le nom exact du répertoire où se trouve le compilateur dépend de l’emplacement du répertoire Windows et la version de « Installé le .NET Framework ».</span><span class="sxs-lookup"><span data-stu-id="2682b-116">The exact name of the directory where the compiler is located depends on the location of the Windows directory and the version of the ".NET Framework" installed.</span></span> <span data-ttu-id="2682b-117">Si vous avez plusieurs versions de « .NET Framework » installé, vous devez déterminer quelle version utiliser (généralement la dernière version).</span><span class="sxs-lookup"><span data-stu-id="2682b-117">If you have more than one version of the ".NET Framework" installed, you must determine which version to use (typically the latest version).</span></span>

6. <span data-ttu-id="2682b-122">Cliquez sur **OK** pour confirmer vos modifications et fermer les boîtes de dialogue.</span><span class="sxs-lookup"><span data-stu-id="2682b-122">Click **OK** to confirm your edits and close the dialog boxes.</span></span>  
  
     <span data-ttu-id="2682b-123">Après avoir modifié la variable d’environnement PATH, vous pouvez exécuter le compilateur Visual Basic à l’invite de commandes Windows à partir de n’importe quel répertoire sur l’ordinateur.</span><span class="sxs-lookup"><span data-stu-id="2682b-123">After you change the PATH environment variable, you can run the Visual Basic compiler at the Windows Command Prompt from any directory on the computer.</span></span>
